Django REST Framework（DRF）

DjangoでAPIを作るための強力な拡張キットのこと

---

Serializer（シリアライザー）

「Python（Django）の世界」と「外の世界（Webブラウザなど）」をつなぐ通訳者をする役割

---

サフィックス
Django REST Framework でいう「サフィックス」とは、URL の末尾に付ける .json や .api などの拡張子のこと

```
/snippets/
通常のリクエスト

/snippets.json
.json サフィックス付きでも同じビューにルーティングされる

/snippets/1/
個別スニペット取得

/snippets/1.json
個別スニペット取得（サフィックス付き）

```

---

GET：データ取得
普通の読み取り

POST：データ作成
新規スニペット作成

HEAD：GET のヘッダのみ
存在チェック、サイズチェック

OPTIONS：使用可能メソッド確認
ブラウザやクライアントの自動チェック

HEADとOPTIONSはAPIが自動で追加してくれる便利な補助メソッドで、
手動で書いた GET / POST / PUT / DELETE とは別に、クライアントが API を扱いやすくするため に存在しているだけ

---

関数ベースビュー (FBV)
通常のPython関数でビューを定義する方法。HTTPリクエストを受け取り、HTTPレスポンスを返す

```
from django.shortcuts import render
from django.http import HttpResponse

def my_view(request):
    if request.method == "POST":
        return HttpResponse("POSTリクエストです")
    return HttpResponse("GETリクエストです")

```

クラスベースビュー (CBV)
Pythonのクラスとしてビューを定義する方法。メソッド（get, postなど）ごとに処理を分けられる

```

from django.views import View
from django.http import HttpResponse

class MyView(View):
    def get(self, request):
        return HttpResponse("GETリクエストです")

    def post(self, request):
        return HttpResponse("POSTリクエストです")

```
